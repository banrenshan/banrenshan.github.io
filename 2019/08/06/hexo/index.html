使用hexo搭建GitPage

2019-08-06

1.环境准备

1.1 软件预装

电脑需要预装下面的软件

  1. nodejs(>6.9)

  2. Git

1.2 安装hexo

npm install -g hexo-cli

2.基本使用

2.1 创建项目

hexo init <folder>
cd <folder>
npm install

2.2 新建文章

hexo new [layout] <title>

新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

$ hexo new "post title with whitespace"

2.3 生成静态文件

$ hexo generate

  1. -d, --deploy:文件生成后立即部署网站

  2. -w, --watch:监视文件变动

该命令可以简写为 hexo g

2.4 发表草稿

 hexo publish [layout] <filename>

2.5 启动服务

hexo server

启动服务器。默认情况下，访问网址为： http://localhost:4000/。

  1. -p, --port:重设端口

  2. -s, --static:只使用静态文件

  3. -l, --log:启动日记记录，使用覆盖记录格式

2.6 部署到git

1.安装插件

npm install hexo-deployer-git --save

2.配置

deploy:
  type: git
  repo: <repository url> #https://bitbucket.org/JohnSmith/johnsmith.bitbucket.io (1)
  branch: [branch] #published (2)
  message: [message] (3)

  1. 库（Repository）地址

  2. 分支名称。如果您使用的是 GitHub 或 GitCafe 的话，程序会尝试自动检测。

  3. 自定义提交信息 (默认为 Site updated: <!--￼0-→)

Note
 这里不需要设置git用户名和密码,是因为会调用git工具,git工具是需要正确配置的用户信息的. 
hexo deploy

  1. -g, --generate:部署之前预先生成静态文件

2.7 清理

hexo clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

3.目录结构

.
├── _config.yml   #网站的配置信息
├── package.json  #npm依赖信息
├── scaffolds     #模版文件夹
├── source   (1)
|   ├── _drafts #草稿目录
|   └── _posts #发布的文章目录
└── themes #主题

  1. 存放用户资源的地方。除 posts文件夹之外，(下划线)开头的文件/文件夹和隐藏的文件都会被忽略。Markdown和HTML文件会被解析并放到public文件夹,其他文件则原样复制到public

4.支持asciidoctor

hexo 默认情况支持markdown,我比较喜欢使用asciidoctor来编写文档,需要做如下修改来开启支持:

  1. 安装插件

npm install hexo-renderer-asciidoc --save

5.使用hexo-theme-indigo模板

git地址:https://github.com/yscoder/hexo-theme-indigo/wiki

1.下载模板

2.初始化项目

hexo init <folder>

3.解压后将模板复制到<folder>/theme下面,修改文件名称为indigo

4.安装依赖

npm install hexo-renderer-less --save
npm install hexo-generator-feed --save
npm install hexo-generator-json-content --save
npm install hexo-helper-qrcode --save
npm install hexo-deployer-git --save
npm install hexo-renderer-asciidoc --save
npm install hexo --save
npm install

5.修改配置

theme: indigo #修改主题
